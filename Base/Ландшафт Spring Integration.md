# Ландшафт Spring Integration

Типичный поток состоит из одного или нескольких компонентов. Рассмотрим их:
- **канал** - передает сообщения между компонентами потока;
- **фильтр** - разрешает или запрещает прохождения сообщения через поток, опираясь на некоторые критерии;
- **преобразователь** - изменяет сообщения или преобразует полезные данные из одного типа в другой;
- **маршрутизатор** - направляет сообщений в один из нескольких каналов, обычно опираясь на заголовки сообщений;
- **сплиттер(разделитель)** - делит входящие сообщения на два или более сообщений, каждое из которых отправляется в свой канал;
- **агрегатор** - противоположность сплиттеру; объединяет несколько сообщений из разных каналов;
- **активатор службы** - передает сообщение некоторому методу Java для обработки, а затем публикует возвращаемое значение в выходном канале;
- **адаптер канала** - соединяет канал с некоторой внешней системой или транспортом; может осуществлять ввод из внешней системой или вывод в неё;
- **шлюз** - передает данные в поток интеграции через интерфейс;

Ранее уже были некоторые из этих компонентов, в примере с записью файлов.

Интерфейс FileWriterGateway играл роль шлюза, через который приложение отправляло текст в файл. Мы также определили преобразователь для преобразования текста сообщения в верхний регистр. А кроме того, в потоке имелось два канала, **textInChannel** и **fileWriterChannel**, соединявшие другие компоненты друг с другом.
